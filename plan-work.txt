Phase 1: ESP32 เป็นแค่ "คนส่งของ" (ทำอันนี้ก่อน เพื่อ Debug)
ESP32: อ่าน Lidar -> แปลง XY -> ส่ง Raw Data ขึ้น Server รัวๆ

Browser: รับจุดมา Plot ลงจอ

ข้อดี: คุณจะเห็นภาพจริงว่า Lidar มันส่ายแค่ไหน Noise เยอะแค่ไหน จะได้จูนถูก

Phase 2: ESP32 เริ่มเป็น "คนคัดแยกขยะ" (Optimization)
ESP32: เพิ่ม Logic Chunking (ตัดก้อน) และ Centroid (หาค่าเฉลี่ย)

ESP32: ส่งข้อมูลเฉพาะตอน "จบก้อน" (ส่งน้อยลง แต่เนื้อๆ เน้นๆ)

Browser: รับข้อมูลที่เป็น "จุดเฉลี่ย" มาแสดงผล (ภาพจะนิ่งขึ้นมาก)

Phase 3: ESP32 เป็น "นักคณิตศาสตร์" (Advance - ถ้าจำเป็น)
ESP32: เปลี่ยนจากส่งจุดเฉลี่ย เป็นส่ง Linear Regression (เส้นตรง)

ESP32: ส่งพิกัด (x1, y1) และ (x2, y2) ของเส้นกำแพง

ข้อดี: แผนที่สวยกริบ เป็นเส้นตรงเป๊ะ
